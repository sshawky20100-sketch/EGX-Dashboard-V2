Prerequisites
①
✅ Check your Python version
Python 3.9 or newer is required. Open a terminal and run:
python --version
Python 3.11.8   ← anything 3.9+ is fine
⚠️
If you see Python 2.x or get "command not found", download Python from python.org/downloads. Make sure to check "Add Python to PATH" during installation on Windows.
②
📦 Verify pip is available
pip --version
pip 24.0 from /usr/lib/python3.11/...   ← any version is fine
ℹ️
On some systems you may need pip3 instead of pip. Both work the same way.
Installation — Step by Step
1
📁 Extract the project zip
Download egx_dashboard.zip and unzip it. You should get a folder called egx_dashboard.
Windows
Mac / Linux
# Right-click the zip → Extract All → choose a folder
# OR in PowerShell:
Expand-Archive egx_dashboard.zip -DestinationPath C:\Projects\
cd C:\Projects\egx_dashboard
📌
Remember the full path to this folder. You'll open a terminal here for all remaining steps.
2
🏠 Create a virtual environment (strongly recommended)
A virtual environment keeps this project's packages separate from your system Python — prevents conflicts.
Windows (PowerShell)
Mac / Linux
# Make sure you are inside the egx_dashboard folder first:
cd C:\Projects\egx_dashboard

# Create virtual environment:
python -m venv venv

# Activate it:
venv\Scripts\activate

# You should see (venv) appear at the start of your prompt:
(venv) C:\Projects\egx_dashboard>
✅
On Mac/Linux the activate command is: source venv/bin/activate
3
📦 Install all dependencies
This installs Streamlit, Plotly, pandas, BeautifulSoup, and all other required packages.
pip install -r requirements.txt

Collecting streamlit>=1.32.0
Collecting pandas>=2.0.0
Collecting plotly>=5.18.0
...
Successfully installed streamlit-1.35.0 pandas-2.2.2 plotly-5.22.0 ✓
⏱️
This takes 2–5 minutes on first run. All packages are downloaded from PyPI. An internet connection is required.
4
🚀 Launch the dashboard
This starts the Streamlit web server on your machine.
streamlit run app.py


  You can now view your Streamlit app in your browser.

  Local URL:   http://localhost:8501
  Network URL: http://192.168.1.x:8501    ← share with others on same WiFi
🌐
Your browser should open automatically. If not, manually navigate to http://localhost:8501
5
📊 First load — what happens
On your very first visit the app automatically:
1. Tries to fetch live quotes from Yahoo Finance for EGX stocks
2. Scrapes financial news from 5 Egyptian/regional sources
3. Runs rule-based sentiment analysis on the news
4. Caches everything to data/cache/ for 15 minutes
5. Displays all 195 stocks across 20 sectors
💡
If scraping is blocked (firewall, rate limit, etc.) the app seamlessly falls back to realistic deterministic simulation data — the dashboard remains fully functional and all analytics work identically.
Optional — AI-Powered Sentiment
★

# (look for the "Anthropic API Key (optional)" text box)

Using the Dashboard
▶
🔄 Refreshing data
The sidebar has two buttons:
• 🔄 Refresh — clears the disk cache and re-fetches everything from sources
• 🗑️ Clear Cache — removes cached files so next page load re-fetches
⏱️
A full refresh makes up to ~80 HTTP requests and takes 30–90 seconds. Data is then cached for 15 minutes so navigation between pages is instant.
▶
📊 Running Technical Analysis
Go to Technical Analysis page → select a stock from the dropdown → adjust the history slider → the chart renders automatically with all indicators.
▶
🎯 Getting a Recommendation
Go to Recommendation Engine → select a stock → click "Generate Recommendation". This runs technical analysis + 1,000 Monte Carlo simulations + sentiment lookup, then shows a combined Buy/Sell/Hold with full reasoning.
▶
📂 Building your Portfolio
Go to Portfolio Tracker → use the left form to add holdings (symbol, quantity, purchase price, date) → the right panel shows real-time P&L, allocation pie chart, and per-holding Monte Carlo risk.

Troubleshooting / FAQ
❓ "ModuleNotFoundError: No module named 'streamlit'"
Your virtual environment is not activated. Run venv\Scripts\activate (Windows) or source venv/bin/activate (Mac/Linux) first, then retry.
❓ "streamlit: command not found"
Try python -m streamlit run app.py instead. This uses the Python-module path directly.
❓ The app loads but shows no stock data
Click 🔄 Refresh in the sidebar. On first load, data fetches automatically, but may need one manual trigger if auto-load timed out.
❓ All stocks show "simulated" source
This is normal if Yahoo Finance is rate-limiting you or if you're offline. The simulated prices are deterministically seeded (same values all day) from real EGX base prices — all analytics work identically.
❓ Technical chart doesn't show for a stock
Some smaller EGX stocks don't have Yahoo Finance history. The app automatically falls back to Geometric Brownian Motion synthetic history, so you'll always get a chart.
❓ Monte Carlo simulation is slow
1,000 simulations typically takes 1–3 seconds. If you chose 5,000 simulations it may take 10–15 seconds. Results are not cached (always fresh). Try 1,000 for speed.
❓ How do I stop the app?
Press Ctrl + C in the terminal window where Streamlit is running. The browser tab will show a disconnect message.
❓ Can I access the dashboard from my phone or another device?
Yes! When Streamlit starts, it shows a Network URL like http://192.168.x.x:8501. Open that on any device connected to the same WiFi network.
❓ How do I add more stocks to the universe?
Open data/egx_universe.py and add a new entry to the EGX_FULL_UNIVERSE list following the format: ("SYMBOL", "Full Company Name", "Sector", "Subsector", base_price, market_cap). The app picks it up automatically on next restart.
